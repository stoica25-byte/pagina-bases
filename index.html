Estructura de un Bloque PL/SQL

PL/SQL es un lenguaje procedimental estructurado en bloques. Un bloque se divide en tres secciones principales: Declaración, Ejecución y Excepciones.

DECLARE
  -- Variables, constantes y cursores locales
  v_salario emp.sal%TYPE;
  v_limite  CONSTANT NUMBER := 3000;
BEGIN
  -- Sentencias ejecutables (SQL y control)
  SELECT sal INTO v_salario FROM emp WHERE empno = 7839;
  
  IF v_salario > v_limite THEN
    DBMS_OUTPUT.PUT_LINE('El salario supera el límite.');
  ELSE
    DBMS_OUTPUT.PUT_LINE('El salario es aceptable.');
  END IF;
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    DBMS_OUTPUT.PUT_LINE('El empleado no existe.');
END;

Uso de Cursores en Oracle

Los cursores permiten procesar fila a fila el conjunto de resultados de una consulta multilínea. En Oracle se declaran en el bloque DECLARE y se controlan con OPEN, FETCH y CLOSE.

DECLARE
  CURSOR c_empleados IS
    SELECT ename, sal FROM emp WHERE deptno = 10;
  v_nom emp.ename%TYPE;
  v_sal emp.sal%TYPE;
BEGIN
  OPEN c_empleados;
  LOOP
    FETCH c_empleados INTO v_nom, v_sal;
    EXIT WHEN c_empleados%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE(v_nom || ' gana: ' || v_sal);
  END LOOP;
  CLOSE c_empleados;
END;

Control de Excepciones

Las excepciones se utilizan para capturar errores de base de datos o lógicos en tiempo de ejecución. Oracle proporciona excepciones predefinidas comunes que puedes capturar por su nombre.

Excepción Oracle Causa Común
NO_DATA_FOUND Un SELECT INTO no devolvió ninguna fila.
TOO_MANY_ROWS Un SELECT INTO devolvió más de una fila.
DUP_VAL_ON_INDEX Intento de insertar una clave primaria o única duplicada.
ZERO_DIVIDE Intento de división por cero. 
EXCEPTION
  WHEN DUP_VAL_ON_INDEX THEN
    DBMS_OUTPUT.PUT_LINE('Error: El registro con esa clave ya existe.');
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Ocurrió un error inesperado.');

DCL: Gestión de Usuarios y Permisos

El Lenguaje de Control de Datos sirve para dar o revocar accesos en el esquema de base de datos. Recuerda la sintaxis básica para crear perfiles, cuotas y otorgar privilegios:

-- 1. Crear un usuario con cuota en tablespace
CREATE USER test_user IDENTIFIED BY pass123
DEFAULT TABLESPACE users
TEMPORARY TABLESPACE temp;

ALTER USER test_user QUOTA 10M ON users;

-- 2. Conceder privilegios de sistema
GRANT CREATE SESSION, CREATE TABLE, CREATE VIEW TO test_user;

-- 3. Conceder privilegios de objeto
GRANT SELECT, INSERT ON scott.emp TO test_user;

-- 4. Quitar privilegios
REVOKE INSERT ON scott.emp FROM test_user;

Vistas, Índices y Secuencias

Estructuras adicionales de Oracle que debes saber definir en la parte práctica de la UD09:

-- Crear o reemplazar una Vista
CREATE OR REPLACE VIEW vista_emp_dept10 AS
  SELECT empno, ename, job, sal 
  FROM emp 
  WHERE deptno = 10;

-- Crear un índice para acelerar búsquedas
CREATE INDEX idx_emp_apellido ON emp(ename);

-- Crear y usar una secuencia autoincremental
CREATE SEQUENCE sec_dept_id
  START WITH 50
  INCREMENT BY 10;

-- Ejemplo de inserción usando la secuencia
INSERT INTO dept VALUES (sec_dept_id.NEXTVAL, 'MKT', 'MADRID');
